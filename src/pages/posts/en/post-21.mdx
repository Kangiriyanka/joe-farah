Because we want to work inside a system where we want to access children of categories (subfolders), we have to first consider what goes on if we work our original structure
Let's say we want to access the Poison Ivy category; we start at the Nature folderWe could simply choose to work with our 1D array and scan the whole array every time we want to get .
Visually, if I start at the Nature category and access its subcategories by tapping/clicking on it, I would need 2 scans of the array to reach Poison Ivy. The result of scanning in this context means executing function that gets the children of a parent category.

Instead, we'll create a children map where the keys are parent_ids (integers) and the values are list of Category objects stored as dicts. This adjacent list is the structure used to build a tree.


```python
def _build_category_tree(categories):
    children_map = defaultdict(list)

    # Building a children map is efficient for executing a DFS.
    for cat in categories:
        children_map[cat.parent_id].append(cat)
   

    def dfs(parent_id):

        if parent_id not in children_map:
            return []
        return [
            {
                **cat.to_json(),
                "children": dfs(cat.id)
            }
            for cat in children_map[parent_id]
        ]

    # Top level categories exist in the None bucket
    return dfs(None)


```
